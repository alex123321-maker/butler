# Codex Playbooks

Small role-oriented entrypoints for Codex Desktop sessions in Butler.

## Build

Use when the task is mostly implementation and verification.

Reference:
- `prompts/butler-build.md`

## Plan

Use when the task changes multiple subsystems or you need an explicit impact map before editing.

Reference:
- `prompts/butler-plan.md`

## Architecture

Use when work may affect service boundaries, ADR-level concerns, or spec alignment.

Reference:
- `prompts/butler-architecture.md`

## Transport

Use when work touches model providers, streaming, provider sessions, or transport contracts.

Reference:
- `prompts/butler-transport.md`

## Tools

Use when work touches Tool Broker, runtime isolation, tool schemas, or browser/HTTP runtime behavior.

Reference:
- `prompts/butler-tools.md`

## Memory

Use when work touches transcript storage, working memory, profile memory, episodic memory, provenance, or retrieval.

Reference:
- `prompts/butler-memory.md`

## Security

Use when work touches secrets, credential refs, approval rules, or auth injection.

Reference:
- `prompts/butler-security.md`

## Doctor

Use when work touches diagnostics, observability, operator health views, or self-inspection.

Reference:
- `prompts/butler-doctor.md`
